# Architecture Decision Records (ADR)

Architecture Decision Records capture the significant technical and product decisions made during the development of Navo.

Their purpose is to explain **why** a decision was made, not just **what** was implemented.

Each ADR is intended to remain stable over time. If a decision changes, a new ADR should be created that supersedes the previous one.

## ADR Structure

Each ADR contains:

- Title
- Status
- Context
- Decision
- Consequences

## Naming

ADRs use sequential numbering.

Examples:

- `0001-intent-first.md`
- `0002-work-is-a-graph.md`
- `0003-planning-and-scheduling.md`

Numbers never change after publication.
