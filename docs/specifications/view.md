# View Specification

## Purpose

Views present the same information in different ways.

Views never own data.

## Available Views

- Intent
- Daily Planner
- Timeline
- Calendar
- Kanban
- List
- Map
- Graph

## Rules

Every View:

- reads from the same underlying model
- preserves relationships
- supports interaction
- never duplicates information

Changing a View never changes the underlying data model.

Views answer different questions while representing the same work.
