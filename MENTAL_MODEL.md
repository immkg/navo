# Mental Model

Navo is built around the belief that software should model the way people think, not the way databases store information.

People do not naturally think in tasks, projects, or folders. They think in outcomes, opportunities, places, people, and decisions.

Navo models these concepts directly.

---

# The Journey Begins With Intent

Everything starts with an intent.

An intent is simply something you want to accomplish.

Examples:

* Buy groceries
* Plan a vacation
* Organize a wedding
* Prepare for interviews

An intent does not require a plan.

It only represents a desired outcome.

---

# Intent Creates Work

As people make progress, work naturally emerges.

Some work is obvious.

Some work is discovered.

Some work only becomes possible after previous work is completed.

Planning is therefore a continuous process of discovery.

---

# Work Exists Within Context

The same work can be performed differently depending on context.

Examples:

* Buy groceries near home.
* Buy groceries while returning from work.
* Buy groceries during the weekly market.

The work is identical.

The context changes the best decision.

---

# Work Forms Relationships

Real work is connected.

One task may depend on another.

One decision may unlock several new actions.

Multiple people may contribute to the same work.

Several intentions may share common work.

Relationships are more important than hierarchy.

---

# Plans Are Routes

A plan is not a checklist.

A plan is a route through work.

Just as navigation software continuously finds the best path to a destination, Navo continuously adapts the path toward an intention.

Different routes may reach the same outcome.

The best route depends on the current context.

---

# Execution Creates Knowledge

Completing work does more than mark progress.

It changes the world.

Execution may:

* Complete an intention.
* Create new work.
* Remove dependencies.
* Change priorities.
* Reveal new opportunities.

Planning therefore never truly ends.

It continuously adapts.

---

# One Model, Many Views

The underlying model never changes.

Different views simply answer different questions.

| View     | Answers                      |
| -------- | ---------------------------- |
| Intent   | What am I trying to achieve? |
| Planner  | What should I do today?      |
| Timeline | What happens over time?      |
| Calendar | When should it happen?       |
| Map      | Where can it happen?         |
| Graph    | How is everything connected? |

Every view represents the same information from a different perspective.

---

# The Navo Loop

```
Intent
    ↓
Work
    ↓
Context
    ↓
Plan
    ↓
Execute
    ↓
Learn
    ↓
New Intent
```

Planning is not a linear process.

It is a continuous cycle of intention, action, learning, and adaptation.
